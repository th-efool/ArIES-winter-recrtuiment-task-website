## ğŸ§  Philosophy

Most programming exercises are solved once and forgotten.

Game development is different.

Here, participants:

* build **interactive systems**
* reason about **state, logic, physics, input, AI**
* learn by **constructing worlds with rules they define**

The goal of this assignment is **not visual polish**.
The goal is **understanding systems deeply enough to explain them**.

> A small, well-understood game beats a large, shallow one.
